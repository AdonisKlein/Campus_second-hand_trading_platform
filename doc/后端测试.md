## 4. 接口测试记录

### 4.1 查询商品列表

- 请求方式：`GET`
- 请求地址：`http://localhost:8080/api/items`
- 请求参数：无
- 预期结果：返回商品列表，请求成功
- 实际结果：返回 `success: true`，接口正常
- 测试结论：通过

- 请求方式：`POST`
- 请求地址：`http://localhost:8080/api/users/login`

## 5. 问题记录与处理

| 问题 | 原因 | 处理结果 |
| --- | --- | --- |
| MySQL 连接失败 | `application.yml` 中数据库密码格式未正确处理 | 将密码用引号包裹，连接成功 |
| 数据库不存在 | 本地未创建 `campus_secondhand` 数据库 | 执行 `schema.sql` 创建数据库和表 |
| 8080 端口占用 | 后端服务或其他程序占用端口 | 关闭占用进程后重新启动 |
| seed 用户登录失败 | 初始用户密码哈希与 BCrypt 不匹配 | 重新生成 BCrypt 密文并更新 `seed.sql` |
| 发送留言返回 405 | Postman 请求方法使用错误 | 改为 `POST /api/messages` 后正常 |
| 导入 seed 出现 VALUES 废弃提醒 | MySQL 新版本不推荐 `VALUES(col)` 写法 | 改为别名写法 `AS new_users` |
