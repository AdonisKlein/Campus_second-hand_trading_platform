# 后端测试说明

## 1. 测试目标

后端测试用于验证 Spring Boot API 的核心业务逻辑是否符合当前需求，包括用户注册登录、邮箱登录、忘记密码重置、登录失败锁定、商品发布与搜索、留言改删、订单创建、订单状态管理和管理员管理。

## 2. 测试技术

| 工具 | 用途 |
|---|---|
| JUnit 5 | 编写和执行测试用例。 |
| Spring Boot Test | 启动 Spring 测试上下文。 |
| MockMvc | 模拟 HTTP 请求测试 Controller。 |
| H2 | 测试环境内存数据库。 |
| BCryptPasswordEncoder | 生成测试用户密码散列。 |

测试类：

```text
backend/src/test/java/com/campus/secondhand/SecondhandApplicationTests.java
```

测试配置使用 `test` profile，测试过程中使用 H2 内存数据库，不依赖本地 MySQL。

## 3. 执行方式

进入后端目录：

```bash
cd backend
```

执行：

```bash
mvn test
```

成功时 Maven 输出：

```text
Tests run: 9, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

## 4. 测试用例说明

### 4.1 `contextLoads`

验证 Spring Boot 应用上下文可以正常加载。

预期结果：

- 应用上下文启动成功。
- Repository、Controller 等 Bean 可正常创建。

### 4.2 `registerLoginAndRejectWrongPassword`

验证用户注册、用户名登录、邮箱登录和错误密码处理。

测试步骤：

1. 写入邮箱验证码。
2. 调用 `/users/register` 注册用户。
3. 使用正确密码调用 `/users/login`。
4. 使用邮箱和正确密码调用 `/users/login`。
5. 使用错误密码调用 `/users/login`。

预期结果：

- 注册成功。
- 正确密码登录成功。
- 邮箱登录成功。
- 错误密码登录失败，并提示剩余尝试次数。

### 4.3 `lockUserAfterThreeFailedLogins`

验证连续 3 次登录失败后账号锁定。

测试步骤：

1. 创建测试用户。
2. 连续两次使用错误密码登录。
3. 第三次使用错误密码登录。
4. 锁定期间使用正确密码登录。

预期结果：

- 前两次失败但未锁定。
- 第三次失败后返回“密码错误次数过多，账号已锁定10分钟”。
- 锁定期间正确密码也不能登录。

### 4.4 `resetPasswordWithVerificationCodeAndLoginWithNewPassword`

验证忘记密码验证码重置和新旧密码登录结果。

测试步骤：

1. 创建带邮箱的测试用户。
2. 写入该邮箱的验证码。
3. 调用 `/users/forgot-password/reset` 重置密码。
4. 使用旧密码调用 `/users/login`。
5. 使用新密码调用 `/users/login`。

预期结果：

- 重置密码接口返回成功。
- 旧密码登录失败。
- 新密码登录成功，并返回用户基本信息。

### 4.5 `publishAndSearchItems`

验证商品发布和关键词搜索。

测试步骤：

1. 创建卖家用户。
2. 调用 `/items` 发布商品。
3. 调用 `/items?keyword=数学` 搜索商品。

预期结果：

- 商品发布成功。
- 搜索结果包含发布的商品。

### 4.6 `sendAndListMessages`

验证留言发送、列表查询、发送者昵称展示。

测试步骤：

1. 创建卖家、买家和商品。
2. 调用 `/messages` 发送留言。
3. 调用 `/messages/item/{itemId}` 查询留言。

预期结果：

- 留言保存成功。
- 留言列表包含刚发送的留言。
- 留言列表返回发送者昵称。

### 4.7 `updateAndDeleteOwnMessageOnly`

验证留言发送者可修改和删除自己的留言，其他用户不能删除。

测试步骤：

1. 创建卖家、买家、其他用户和商品。
2. 买家发送留言。
3. 买家修改留言内容。
4. 其他用户尝试删除留言。
5. 买家删除留言。
6. 查询商品留言列表。

预期结果：

- 买家修改留言成功。
- 其他用户删除留言失败。
- 买家删除留言成功。
- 留言列表变为空。

### 4.8 `adminCanManageUsersMessagesAndItems`

验证管理员管理普通用户、留言和商品。

测试步骤：

1. 创建管理员、普通用户和商品。
2. 普通用户发送留言。
3. 管理员禁用普通用户。
4. 被禁用用户尝试登录。
5. 管理员删除留言。
6. 管理员下架商品。
7. 查询首页商品列表。

预期结果：

- 普通用户状态变为 `DISABLED`。
- 被禁用用户不能登录。
- 留言删除成功。
- 商品状态变为 `REMOVED`。
- 首页不再展示被下架商品。

### 4.9 `createOrderMarksItemSoldAndValidatesStatus`

验证订单创建、重复下单拦截、订单状态校验和商品售出联动。

测试步骤：

1. 创建卖家、买家和商品。
2. 调用 `/orders` 创建订单。
3. 再次对同一商品创建订单。
4. 调用 `/orders/{id}/status` 更新为 `CONFIRMED`。
5. 尝试更新为非法状态 `WRONG`。
6. 查询商品详情。

预期结果：

- 首次创建订单成功，订单状态为 `CREATED`。
- 重复下单失败。
- 合法状态更新成功。
- 非法状态更新失败。
- 商品状态变为 `SOLD`。

## 5. 测试数据清理

每个测试前执行清理逻辑：

```java
orderRepository.deleteAll();
messageRepository.deleteAll();
itemRepository.deleteAll();
userRepository.deleteAll();
emailVerificationRepository.deleteAll();
```

这样可以保证各测试用例互不影响。

## 6. 测试结论

当前后端自动化测试覆盖了主要核心流程。测试通过说明后端核心接口和业务规则可正常工作。
