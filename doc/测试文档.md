# 校园二手交易平台测试文档

## 1. 测试概述

本测试报告依据当前项目实现状态编写，测试对象为校园二手交易平台的后端接口、前端主要页面流程和数据库初始化脚本。后端接口统一挂载在 `/api` 上，自动化测试通过 MockMvc 直接测试 Controller 映射。

当前系统实现内容包括：

- 用户注册、用户名登录、邮箱登录、邮箱验证码。
- 忘记密码和邮箱验证码重置密码。
- 登录失败 3 次临时锁定 10 分钟。
- 个人资料查看和修改。
- 商品浏览、搜索、发布、详情查看。
- 商品留言、留言修改和删除。
- 订单创建、订单列表、订单状态更新。
- 商品下单后状态变为 `SOLD`。
- 管理员禁用/恢复普通用户、删除留言、下架/恢复商品。

## 7. 数据库测试

执行 `database/schema.sql` 后，应创建：

```text
users
items
messages
trade_orders
email_verification
```

执行 `database/seed.sql` 后，应插入演示用户和商品。

重点字段检查：

- `users.password_hash` 用于存储 BCrypt 密码。
- `users.role` 用于区分普通用户和管理员。
- `users.status` 用于区分正常和禁用账号。
- `users.login_failed_count` 用于记录登录失败次数。
- `users.locked_until` 用于记录账号锁定截止时间。
- `items.status` 用于区分 `ON_SALE`、`SOLD` 和 `REMOVED`。
- `trade_orders.status` 用于记录订单状态。

## 8. 发现的问题和处理

| 问题 | 当前处理 |
|---|---|
| 旧数据库表结构与当前版本不一致 | README 和安装手册说明全新部署时先删除旧库，再执行 `schema.sql` 和 `seed.sql`。 |
| 图片上传需求与当前实现不完全一致 | 当前版本明确为“图片地址”输入，不支持本地文件上传。 |
| 邮箱验证码依赖 SMTP 配置 | 安装手册说明需要配置 `spring.mail`。 |
| 前端登录态较简单 | 当前使用 `localStorage` 保存用户基本信息，适合课程项目演示。 |
| 后端未启用正式 Token 鉴权 | 当前接口主要依赖请求中的用户 ID 和角色字段完成课程项目演示，生产环境需补充认证授权机制。 |

## 9. 测试结论

当前项目后端自动化测试全部通过，前端核心流程与数据库脚本符合课程项目的基本验收要求。系统能够完成校园二手交易平台的主要业务闭环：注册登录、忘记密码、商品发布与浏览、留言沟通、创建订单、订单管理、个人资料维护和管理员管理。
