# 校园二手交易平台设计文档

## 1. 系统架构

系统采用简易前后端分离架构：

```text
浏览器
  |
  | 静态页面 HTML/CSS/JavaScript
  v
frontend/
  |
  | HTTP JSON 请求
  v
Spring Boot 后端 /api
  |
  | Spring Data JPA
  v
MySQL campus_secondhand
```

前端页面存放在 `frontend/` 目录，可直接打开 HTML 文件，也可通过静态服务器访问。后端项目存放在 `backend/` 目录，使用 Spring Boot 3.5.14，默认端口为 `8080`，统一接口前缀为 `/api`。数据库脚本位于 `database/` 目录。

## 2. 技术选型

| 层次 | 技术 |
|---|---|
| 前端 | HTML、CSS、JavaScript |
| 后端 | Java 24、Spring Boot、Spring Web、Spring Data JPA |
| 数据库 | MySQL 8.x |
| 安全相关 | Spring Security Crypto BCrypt |
| 邮件 | Spring Boot Mail |
| 测试 | JUnit 5、Spring Boot Test、MockMvc、H2 |
| 构建 | Maven |

## 4. 后端模块设计

### 4.1 用户模块

负责注册、用户名/邮箱登录、忘记密码、邮箱验证码发送、用户资料查询与修改、用户名/邮箱重复检查。

主要设计：

- `users` 表保存账号、密码散列、昵称、手机号、邮箱、角色、账号状态、登录失败次数和锁定时间。
- 注册时调用验证码服务校验邮箱验证码。
- 密码通过 BCrypt 加密后保存。
- 登录时检查锁定状态；连续 3 次失败后锁定 10 分钟；成功登录后清空失败计数。
- 登录时拒绝已被管理员禁用的账号。
- 忘记密码时先校验注册邮箱，再通过邮箱验证码重置密码。
- 修改邮箱时要求新邮箱验证码。

### 4.2 商品模块

负责商品发布、商品列表查询、关键词搜索、分类筛选和商品详情。

主要设计：

- `items` 表保存商品标题、分类、价格、描述、图片地址、卖家 ID 和状态。
- 商品状态默认为 `ON_SALE`。
- 商品下单成功后状态更新为 `SOLD`。
- 首页列表只展示可售商品。
- 管理员可将商品状态改为 `REMOVED` 进行下架，也可恢复为 `ON_SALE`。

### 4.3 留言模块

负责商品详情页的留言发送和留言列表查询。

主要设计：

- `messages` 表记录商品 ID、发送方、接收方、内容和创建时间。
- 留言按商品聚合展示。
- 留言发送需要前端提供当前登录用户 ID。
- 留言列表返回发送者昵称和发布时间。
- 留言发送者可修改或删除自己的留言，管理员可删除任意留言。

### 4.4 订单模块

负责订单创建、用户订单列表查询和订单状态更新。

主要设计：

- `trade_orders` 表保存商品、买家、卖家、状态和创建时间。
- 创建订单前检查买家、卖家、商品是否存在。
- 买家不能购买自己的商品。
- 商品已下单或已售时不能重复创建有效订单。
- 允许订单状态：`CREATED`、`CONFIRMED`、`COMPLETED`、`CANCELLED`。

### 4.5 邮箱验证码模块

负责生成、保存、发送和校验邮箱验证码。

主要设计：

- `email_verification` 表保存邮箱、验证码、创建时间、过期时间、尝试次数和是否使用。
- 验证码用于注册、修改邮箱和重置密码。
- 邮件发送依赖 `application.yml` 中的 `spring.mail` 配置。

### 4.6 管理员模块

负责管理中心接口，包括普通用户状态管理、留言管理和商品上下架。

主要设计：

- 管理接口位于 `/api/admin`。
- 后端根据请求中的 `adminId` 查询用户并校验 `role = ADMIN`。
- 管理员可禁用或恢复普通用户，不允许修改管理员账号。
- 管理员可删除任意留言。
- 管理员可将商品状态设置为 `REMOVED` 或恢复为 `ON_SALE`。

## 5. 前端页面设计

| 页面 | 文件 | 功能 |
|---|---|---|
| 首页 | `frontend/index.html` | 商品列表、关键词搜索、分类筛选。 |
| 注册页 | `frontend/register.html` | 用户注册和邮箱验证码发送。 |
| 详情页 | `frontend/detail.html` | 商品详情、留言列表、发送留言、创建订单。 |
| 发布页 | `frontend/publish.html` | 发布商品，图片字段为图片地址。 |
| 订单页 | `frontend/orders.html` | 查看相关订单并更新订单状态。 |
| 个人中心 | `frontend/profile.html` | 登录、忘记密码、资料查看、资料修改、退出登录。 |
| 管理中心 | `frontend/admin.html` | 普通用户管理、留言管理、商品管理。 |

前端公共请求封装位于 `frontend/assets/js/api.js`，默认 API 地址为：

```javascript
const API_BASE = "http://localhost:8080/api";
```

## 6. 接口设计

接口统一返回 `ApiResponse` 结构：

```json
{
  "success": true,
  "message": "success",
  "data": {}
}
```

### 6.1 用户接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `POST` | `/api/users/send-verification` | 发送邮箱验证码。 |
| `POST` | `/api/users/register` | 注册用户。 |
| `POST` | `/api/users/login` | 用户登录。 |
| `POST` | `/api/users/forgot-password/send-code` | 忘记密码时发送邮箱验证码。 |
| `POST` | `/api/users/forgot-password/reset` | 校验验证码并重置密码。 |
| `GET` | `/api/users/{id}` | 查询用户资料。 |
| `PUT` | `/api/users/{id}` | 修改用户资料。 |
| `GET` | `/api/users/check` | 检查用户名或邮箱是否存在。 |

### 6.2 商品接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/items` | 商品列表，可带 `keyword`、`category` 参数。 |
| `GET` | `/api/items/{id}` | 商品详情。 |
| `POST` | `/api/items` | 发布商品。 |

### 6.3 留言接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/messages/item/{itemId}` | 查询某商品留言。 |
| `POST` | `/api/messages` | 发送留言。 |
| `PUT` | `/api/messages/{id}` | 修改自己的留言。 |
| `DELETE` | `/api/messages/{id}` | 删除自己的留言。 |

### 6.4 订单接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/orders?userId={id}` | 查询用户作为买家或卖家的订单。 |
| `POST` | `/api/orders` | 创建订单。 |
| `PUT` | `/api/orders/{id}/status` | 更新订单状态。 |

### 6.5 管理员接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `GET` | `/api/admin/users?adminId={id}` | 查询普通用户列表。 |
| `PUT` | `/api/admin/users/{id}/status` | 禁用或恢复普通用户。 |
| `GET` | `/api/admin/messages?adminId={id}` | 查询全部留言。 |
| `DELETE` | `/api/admin/messages/{id}` | 管理员删除留言。 |
| `GET` | `/api/admin/items?adminId={id}` | 查询全部商品。 |
| `PUT` | `/api/admin/items/{id}/status` | 下架或恢复商品。 |

## 8. 安全设计

- 密码使用 BCrypt 加密。
- 注册、修改邮箱和重置密码使用验证码。
- 登录失败计数保存在数据库中，达到 3 次后锁定账号 10 分钟。
- 锁定期间直接拒绝登录请求。
- 前端登录态仅保存用户 ID、用户名、昵称、手机号、邮箱、角色和状态等非密码信息。
- 管理员接口在后端校验管理员角色；当前项目未引入 Token，适合作为课程项目的简易权限模型。

## 9. 当前设计限制

- 未实现真实文件上传，商品图片通过图片地址展示。
- 未实现基于 Token 的认证授权，前端依赖 `localStorage` 保存当前用户。
- 留言没有私信会话页，仅在商品详情页展示。
- 订单状态按钮没有按买家/卖家做权限区分。
