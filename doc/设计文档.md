# 校园二手交易平台设计文档

## 1. 架构设计

系统采用简易前后端分离架构：

- 前端负责页面展示和用户交互。
- 后端提供 REST API。
- MySQL 保存用户、物品、留言、订单数据。

## 2. 模块划分

| 模块 | 后端包 | 主要职责 |
| --- | --- | --- |
| 用户模块 | `user` | 注册、登录、个人资料 |
| 物品模块 | `item` | 发布、查询、详情 |
| 沟通模块 | `message` | 留言发送和查询 |
| 订单模块 | `order` | 订单创建、查询、状态维护 |

## 3. 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/users/register` | 用户注册 |
| POST | `/api/users/login` | 用户登录 |
| GET | `/api/items` | 物品列表、分类查询、关键词搜索 |
| GET | `/api/items/{id}` | 物品详情 |
| POST | `/api/items` | 发布物品 |
| GET | `/api/messages/item/{itemId}` | 查询物品留言 |
| POST | `/api/messages` | 发送留言 |
| GET | `/api/orders?userId=1` | 查询用户相关订单 |
| POST | `/api/orders` | 创建订单 |
| PUT | `/api/orders/{id}/status` | 更新订单状态 |
