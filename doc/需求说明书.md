# 校园二手交易平台需求说明书

## 1. 项目概述

校园二手交易平台面向校园学生，提供二手物品浏览、搜索、发布、留言沟通、下单、订单管理、个人资料维护和管理员管理功能。系统采用前后端分离的简易架构，前端为静态 HTML/CSS/JavaScript 页面，后端为 Spring Boot REST API，数据库使用 MySQL。

本文档依据当前项目实现状态编写，用于说明系统已经实现和需要满足的功能、数据、安全、运行环境及测试要求。

## 2. 用户角色

| 角色 | 说明 |
|---|---|
| 游客 | 未登录用户，可浏览首页商品、搜索商品、查看商品详情和留言列表。 |
| 注册用户 | 已注册并登录的用户，可发布商品、留言、创建订单、管理订单和维护个人资料。 |
| 卖家 | 发布商品的注册用户，可接收留言、查看与自己相关的订单并更新订单状态。 |
| 买家 | 对商品感兴趣的注册用户，可留言、创建订单、查看与自己相关的订单。 |
| 管理员 | 具有管理中心权限，可禁用或恢复普通用户、删除留言、下架或重新上架商品。 |

## 3. 功能需求

### 3.1 用户注册

用户可通过注册页面创建账号。注册信息包括用户名、密码、昵称、邮箱和邮箱验证码。

具体要求：

- 用户名不能为空，且必须唯一。
- 密码必须为 6 到 12 位字母和数字组合。
- 注册前需要向邮箱发送验证码。
- 注册时必须提交正确且未过期的邮箱验证码。
- 密码使用 BCrypt 加密后存储，不保存明文密码。
- 注册成功后返回用户基本信息，不返回密码散列。

### 3.2 用户登录

用户可使用用户名和密码登录，也可使用邮箱和密码登录系统。

具体要求：

- 登录成功后，前端将用户基本信息保存到 `localStorage`，用于后续发布、留言、下单等操作。
- 登录失败时返回错误提示。
- 同一账号连续 3 次登录失败后，账号临时锁定 10 分钟。
- 锁定期间即使密码正确，也不允许登录。
- 登录成功后清空失败次数和锁定时间。
- 被管理员禁用的账号不能登录。

### 3.3 忘记密码与密码重置

用户忘记密码时，可在个人中心登录区域进入忘记密码流程，通过注册邮箱接收验证码并设置新密码。

具体要求：

- 重置密码前必须输入已注册邮箱。
- 系统仅对已注册邮箱发送忘记密码验证码。
- 重置密码时必须提交正确且未过期的邮箱验证码。
- 新密码必须为 6 到 12 位字母和数字组合。
- 重置成功后使用 BCrypt 加密保存新密码，旧密码不再可用。

### 3.4 个人资料管理

登录用户可在个人中心查看和修改个人资料。

支持字段：

- 用户名：只读。
- 昵称：可修改。
- 手机号：可修改。
- 邮箱：可修改。

修改邮箱时需要发送并校验新的邮箱验证码。系统需要检查新邮箱是否已被其他用户使用。

### 3.5 商品浏览与搜索

首页展示当前可售商品列表。

具体要求：

- 展示商品标题、分类、价格、描述和图片。
- 支持按关键词搜索商品。
- 支持按分类筛选商品。
- 商品列表默认仅展示状态为 `ON_SALE` 的商品。
- 点击商品卡片进入详情页。

### 3.6 商品发布

登录用户可发布二手商品。

发布字段：

- 标题：必填，最长 120 字符。
- 分类：必填，支持书籍、生活用品、电子产品、其他。
- 价格：必填，非负数，支持小数。
- 图片地址：选填，用于展示商品图片。
- 描述：选填。

当前实现使用图片地址字段 `imageUrl`，不包含本地文件上传流程。

### 3.7 商品详情

用户可查看商品详情。

展示内容：

- 商品图片。
- 标题。
- 分类。
- 价格。
- 描述。
- 下单按钮。
- 留言列表和留言输入框。

### 3.8 留言沟通

用户可在商品详情页查看留言并发送留言。

具体要求：

- 游客可查看留言列表。
- 登录用户可发送留言。
- 留言发送者可修改或删除自己的留言。
- 管理员可删除任意留言。
- 留言与商品、发送方、接收方关联。
- 当前留言展示以商品为维度，显示发送用户昵称、发布时间和留言内容。

### 3.9 订单创建与管理

登录用户可在商品详情页创建订单。

具体要求：

- 买家不能购买自己发布的商品。
- 创建订单时校验买家、卖家、商品是否存在。
- 创建订单时校验卖家是否与商品信息匹配。
- 商品非 `ON_SALE` 状态时不能再次下单。
- 订单创建成功后，商品状态更新为 `SOLD`。
- 用户可在订单管理页查看自己作为买家或卖家的订单。
- 支持更新订单状态：`CONFIRMED`、`COMPLETED`、`CANCELLED`。
- 不合法订单状态应被拒绝。

### 3.10 管理员管理

管理员登录后可进入管理中心。

具体要求：

- 可查看普通用户列表。
- 可禁用或恢复普通用户账号。
- 被禁用用户不能登录、发布商品、发送留言或创建订单。
- 可查看全部留言并删除不合规留言。
- 可查看全部商品，并将商品状态更新为 `REMOVED` 下架，或恢复为 `ON_SALE`。

### 3.11 数据初始化

系统提供数据库脚本：

- `database/schema.sql`：创建数据库和表。
- `database/seed.sql`：导入演示用户和商品数据。

演示用户和管理员密码散列已写入脚本。管理员账号为 `admin`，密码为 `abc123`。

## 4. 非功能需求

### 4.1 运行环境

- JDK 24.0.2。
- Maven 3.8 或以上。
- MySQL 8.x。
- 现代浏览器，如 Chrome、Edge、Firefox。

### 4.2 性能要求

课程测试环境下，系统应支持小规模并发访问，满足 10 到 20 人基本浏览、发布、留言和下单操作。常规页面加载和接口响应应保持在用户可接受范围内。

### 4.3 安全要求

- 密码必须加密存储。
- 注册、修改邮箱和重置密码需要邮箱验证码。
- 用户名和邮箱需要唯一性检查。
- 登录失败达到 3 次后临时锁定账号 10 分钟。
- 管理员接口需要校验当前用户角色为 `ADMIN`。
- 前端只保存用户基本信息和角色状态，不保存密码。

### 4.4 可维护性要求

- 后端按用户、商品、留言、订单等模块组织代码。
- 前端按页面拆分 HTML 和 JS。
- 数据库脚本集中存放在 `database/` 目录。
- 文档集中存放在 `doc/` 目录。

## 5. 数据需求

系统使用以下核心数据表：

| 表名 | 用途 |
|---|---|
| `users` | 用户账号、资料、角色、状态、登录失败次数和锁定时间。 |
| `items` | 商品信息、价格、分类、图片地址、卖家和销售状态。 |
| `messages` | 商品留言信息。 |
| `trade_orders` | 订单信息和订单状态。 |
| `email_verification` | 邮箱验证码、过期时间、尝试次数和使用状态。 |

## 6. 当前实现边界

当前版本已实现课程作业所需的核心交易流程，但保留以下简化点：

- 图片使用 URL 或静态资源路径，不提供真实文件上传。
- 前端登录态使用 `localStorage` 保存用户基本信息，没有引入 Token 或 Session 权限体系。
- 管理员权限为课程项目级简易校验，后端根据传入管理员 ID 校验角色。
- 留言按商品维度展示，没有独立会话列表。
- 订单状态按钮对买家和卖家未做细粒度权限区分。
- 邮件服务需要在 `application.yml` 中配置 SMTP 账号后才能真实发送。
