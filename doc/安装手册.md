# 校园二手交易平台安装手册

## 1. 安装目标

本文档说明如何在一台新的 Windows 设备上部署并测试校园二手交易平台。项目由 Spring Boot 后端、MySQL 数据库和静态前端页面组成。

## 2. 环境准备

### 2.1 必需软件

| 软件 | 建议版本 | 用途 |
|---|---|---|
| JDK | 17 或以上 | 运行后端服务。 |
| Maven | 3.8 或以上 | 构建和启动后端项目。 |
| MySQL | 8.x | 保存用户、商品、留言、订单等数据。 |
| 浏览器 | Chrome、Edge、Firefox | 访问前端页面。 |

### 2.2 检查命令

在命令行执行：

```bash
java -version
mvn -version
mysql --version
```

如果能正常显示版本信息，说明基础环境可用。

## 3. 获取项目

将项目复制或拉取到本机，例如：

```text
D:/Code/Software/bigwork/Campus_second-hand_trading_platform
```

项目根目录应包含：

```text
backend/
database/
doc/
frontend/
README.md
```

## 4. 初始化数据库

### 4.1 登录 MySQL

```bash
mysql -u root -p
```

输入 MySQL root 密码后进入 MySQL 控制台。

### 4.2 执行建库建表脚本

如果当前命令行位于项目根目录，可执行：

```sql
source database/schema.sql;
```

Windows 上也可以使用绝对路径：

```sql
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/schema.sql;
```

注意：MySQL 的 `source` 命令建议使用 `/` 分隔路径。

### 4.3 导入测试数据

如需导入演示用户和商品数据，执行：

```sql
source database/seed.sql;
```

或使用绝对路径：

```sql
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/seed.sql;
```

### 4.4 检查结果

```sql
SHOW DATABASES;
USE campus_secondhand;
SHOW TABLES;
```

应看到以下表：

```text
users
items
messages
trade_orders
email_verification
```

### 4.5 已有数据库升级

如果旧版本已经创建过 `users` 表，但缺少登录锁定字段，请执行：

```sql
ALTER TABLE users
ADD COLUMN login_failed_count INT NOT NULL DEFAULT 0,
ADD COLUMN locked_until DATETIME NULL;
```

## 5. 配置后端

后端配置文件：

```text
backend/src/main/resources/application.yml
```

重点检查数据库配置：

```yaml
spring:
    datasource:
        url: jdbc:mysql://localhost:3306/campus_secondhand?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
        username: root
        password: "你的MySQL密码"
```

默认服务配置：

```yaml
server:
    port: 8080
    servlet:
        context-path: /api
```

后端启动后接口地址为：

```text
http://localhost:8080/api
```

### 5.1 邮箱配置

系统注册和修改邮箱会调用邮箱验证码接口。邮件配置位于：

```yaml
spring:
    mail:
        host: smtp.mailtrap.io
        port: 2525
        username: YOUR_MAILTRAP_USERNAME
        password: YOUR_MAILTRAP_PASSWORD
```

如需真实发送邮件，请替换为可用 SMTP 服务账号。测试环境中也可结合后端控制台输出或测试数据进行验证。

## 6. 启动后端

进入后端目录：

```bash
cd backend
```

启动：

```bash
mvn spring-boot:run
```

看到 Spring Boot 启动完成且无数据库连接错误，即表示后端运行成功。

## 7. 配置前端

前端接口地址配置文件：

```text
frontend/assets/js/api.js
```

默认配置：

```javascript
const API_BASE = "http://localhost:8080/api";
```

如果前端和后端在同一台设备运行，可保持不变。

如果在另一台设备访问后端，需要将 `localhost` 改成后端设备的局域网 IP，例如：

```javascript
const API_BASE = "http://192.168.1.10:8080/api";
```

## 8. 启动或打开前端

方式一：直接打开：

```text
frontend/index.html
```

方式二：在 `frontend/` 目录启动静态服务器：

```bash
cd frontend
python -m http.server 5500
```

访问：

```text
http://localhost:5500
```

局域网访问时将 `localhost` 替换为前端所在设备的 IP。

## 9. 验证安装

建议按以下顺序验证：

1. 首页能打开并显示商品列表。
2. 可按关键词或分类搜索商品。
3. 可注册用户并登录。
4. 登录后可发布商品。
5. 可进入详情页留言。
6. 可创建订单。
7. 可在订单管理页查看订单并更新状态。
8. 可在个人中心修改昵称、手机号、邮箱。
9. 连续 3 次输入错误密码后，账号会锁定 10 分钟。

## 10. 常见问题

### 10.1 数据库连接失败

检查：

- MySQL 是否启动。
- `campus_secondhand` 数据库是否已创建。
- `application.yml` 中用户名和密码是否正确。
- MySQL 端口是否为 3306。

### 10.2 前端接口请求失败

检查：

- 后端是否启动。
- `frontend/assets/js/api.js` 中 `API_BASE` 是否正确。
- 浏览器 Network 面板中的请求地址是否指向正确 IP 和端口。

### 10.3 其他设备无法访问后端

检查：

- 两台设备是否在同一局域网。
- 后端设备防火墙是否允许 8080 端口。
- 前端 `API_BASE` 是否仍写为 `localhost`。

### 10.4 邮箱验证码发送失败

检查：

- `spring.mail` 是否配置为真实可用的 SMTP 服务。
- SMTP 用户名、密码、端口是否正确。
- 网络是否允许访问邮件服务器。

### 10.5 商品图片不显示

当前系统保存的是图片地址。请确认输入的图片地址可被浏览器访问，例如：

```text
assets/images/book.svg
```

# 校园二手交易平台用户手册

## 1. 系统入口

打开前端首页：

```text
frontend/index.html
```

或通过静态服务器访问：

```text
http://localhost:5500
```

页面顶部导航包含：

- 首页：点击“校园二手交易”返回商品列表。
- 发布物品：进入商品发布页面。
- 订单管理：查看和处理订单。
- 个人中心：登录、查看和修改个人资料。

## 2. 浏览商品

进入首页后，系统会自动加载当前可售商品。

商品卡片显示：

- 商品图片。
- 分类。
- 标题。
- 价格。
- 简短描述。

点击商品卡片可进入商品详情页。

## 3. 搜索和筛选商品

首页提供关键词和分类筛选。

操作步骤：

1. 在关键词输入框输入商品名称关键字，例如“教材”。
2. 在分类下拉框选择分类，例如“书籍”。
3. 点击搜索按钮。
4. 页面刷新商品列表，显示符合条件的商品。

## 4. 注册账号

进入注册页面：

```text
frontend/register.html
```

注册步骤：

1. 输入用户名。
2. 输入 6 到 12 位字母和数字组合的密码。
3. 输入昵称。
4. 输入邮箱。
5. 点击发送验证码。
6. 查看邮箱或后端测试输出，获取验证码。
7. 输入验证码并提交注册。

注意事项：

- 用户名不能重复。
- 邮箱不能重复。
- 验证码错误或过期时注册失败。

## 5. 登录账号

进入个人中心：

```text
frontend/profile.html
```

登录步骤：

1. 输入用户名。
2. 输入密码。
3. 点击登录。

登录成功后，系统显示个人资料。前端会保存当前用户基本信息，用于发布、留言、下单等操作。

登录安全规则：

- 密码错误会提示剩余尝试次数。
- 连续 3 次登录失败后，账号锁定 10 分钟。
- 锁定期间即使输入正确密码也不能登录。

## 6. 修改个人资料

登录后进入个人中心。

操作步骤：

1. 点击“修改资料”。
2. 修改昵称、手机号或邮箱。
3. 如果修改邮箱，需要先发送邮箱验证码。
4. 输入验证码。
5. 点击保存资料。

保存成功后，页面会显示最新资料。

## 7. 发布商品

进入发布页面：

```text
frontend/publish.html
```

发布前需要先登录。未登录用户进入发布页会被提示并跳转到个人中心。

发布步骤：

1. 输入商品标题。
2. 选择分类：书籍、生活用品、电子产品、其他。
3. 输入价格。
4. 输入图片地址，例如 `assets/images/book.svg`。
5. 输入商品描述。
6. 点击提交发布。

发布成功后跳转回首页。

说明：当前版本使用图片地址，不支持从本地选择文件上传。

## 8. 查看商品详情

在首页点击商品卡片进入详情页。

详情页包含：

- 商品图片。
- 商品标题。
- 分类。
- 价格。
- 描述。
- 创建订单按钮。
- 留言沟通区域。

## 9. 留言沟通

详情页右侧为留言沟通区域。

查看留言：

- 游客和登录用户都可以查看留言。

发送留言：

1. 登录账号。
2. 进入商品详情页。
3. 在留言框输入内容。
4. 点击发送留言。
5. 留言成功后，留言列表自动刷新。

## 10. 创建订单

登录用户可在商品详情页点击“创建订单”。

创建规则：

- 不能购买自己发布的商品。
- 商品已被下单或已售出时不能重复下单。
- 创建成功后商品状态变为 `SOLD`。

## 11. 管理订单

进入订单管理页：

```text
frontend/orders.html
```

页面会显示当前用户作为买家或卖家的订单。

订单信息包括：

- 订单编号。
- 物品名称。
- 价格。
- 买家。
- 卖家。
- 状态。

支持操作：

- 确认：将状态更新为 `CONFIRMED`。
- 完成：将状态更新为 `COMPLETED`。
- 取消：将状态更新为 `CANCELLED`。

## 12. 退出登录

进入个人中心，点击“退出登录”，确认后退出当前账号。退出后需要重新登录才能发布商品、留言或下单。

## 13. 常见提示

| 提示 | 含义 |
|---|---|
| 请先登录后再发布物品 | 当前没有登录，需要进入个人中心登录。 |
| 请先登录后留言 | 留言功能需要登录。 |
| 请先登录后下单 | 创建订单需要登录。 |
| 物品已被下单或售出 | 商品已不可再次创建订单。 |
| 账号已被临时锁定 | 连续输错密码达到 3 次，需要等待 10 分钟。 |
