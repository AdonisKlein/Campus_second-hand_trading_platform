# 校园二手交易平台部署文档

## 1. 部署目标

本文档说明如何在一台新的 Windows 设备上部署并测试校园二手交易平台。项目由 Spring Boot 后端、MySQL 数据库和静态前端页面组成，前端无需打包构建，后端通过 Maven 启动。

## 2. 环境准备

### 2.1 必需软件

| 软件 | 建议版本 | 用途 |
|---|---|---|
| JDK | 24.0.2 | 运行后端服务。 |
| Maven | 3.8 或以上 | 构建和启动后端项目。 |
| MySQL | 8.x | 保存用户、商品、留言、订单等数据。 |
| 浏览器 | Chrome、Edge、Firefox | 访问前端页面。 |
| Python | 3.x，可选 | 启动前端静态服务器。 |

### 2.2 检查命令

在命令行执行：

```bash
java -version
mvn -version
mysql --version
```

如果能正常显示版本信息，说明基础环境可用。

## 3. 获取项目

将项目复制或拉取到本机，例如：

```text
D:/Code/Software/bigwork/Campus_second-hand_trading_platform
```

项目根目录应包含：

```text
backend/
database/
doc/
frontend/
README.md
```

## 4. 初始化数据库

### 4.1 登录 MySQL

```bash
mysql -u root -p
```

输入 MySQL root 密码后进入 MySQL 控制台。

### 4.2 执行建库建表脚本

如果当前命令行位于项目根目录，可执行：

```sql
source database/schema.sql;
```

Windows 上也可以使用绝对路径：

```sql
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/schema.sql;
```

注意：MySQL 的 `source` 命令建议使用 `/` 分隔路径。

### 4.3 导入测试数据

如需导入演示用户和商品数据，执行：

```sql
source database/seed.sql;
```

或使用绝对路径：

```sql
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/seed.sql;
```

演示管理员账号：

```text
用户名：admin
密码：abc123
```

### 4.4 检查结果

```sql
SHOW DATABASES;
USE campus_secondhand;
SHOW TABLES;
```

应看到以下表：

```text
users
items
messages
trade_orders
email_verification
```

### 4.5 旧数据库处理

为保证一次完成部署，建议使用全新的 `campus_secondhand` 数据库。如果本机曾经运行过旧版本项目，请先备份需要保留的数据，然后在 MySQL 控制台执行：

```sql
DROP DATABASE IF EXISTS campus_secondhand;
```

再依次执行：

```sql
source database/schema.sql;
source database/seed.sql;
```

Windows 上使用绝对路径示例：

```sql
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/schema.sql;
source D:/Code/Software/bigwork/Campus_second-hand_trading_platform/database/seed.sql;
```

## 5. 配置后端

后端配置文件：

```text
backend/src/main/resources/application.yml
```

重点检查数据库配置：

```yaml
spring:
    datasource:
        url: jdbc:mysql://localhost:3306/campus_secondhand?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
        username: root
        password: "你的MySQL密码"
```

默认服务配置：

```yaml
server:
    port: 8080
    servlet:
        context-path: /api
```

后端启动后接口地址为：

```text
http://localhost:8080/api
```

### 5.1 邮箱配置

系统注册、修改邮箱和忘记密码重置会调用邮箱验证码接口。邮件配置位于：

```yaml
spring:
    mail:
        host: smtp.mailtrap.io
        port: 2525
        username: YOUR_MAILTRAP_USERNAME
        password: YOUR_MAILTRAP_PASSWORD
```

如需真实发送邮件，请替换为可用 SMTP 服务账号。测试环境中也可结合后端控制台输出或测试数据进行验证。

当前注册、修改邮箱和忘记密码流程都依赖验证码接口；如果 SMTP 未配置成功，相关流程会提示发送失败。

## 6. 启动后端

进入后端目录：

```bash
cd backend
```

启动：

```bash
mvn spring-boot:run
```

看到 Spring Boot 启动完成且无数据库连接错误，即表示后端运行成功。

## 7. 配置前端

前端接口地址配置文件：

```text
frontend/assets/js/api.js
```

默认配置：

```javascript
const API_BASE = "http://localhost:8080/api";
```

如果前端和后端在同一台设备运行，可保持不变。

如果在另一台设备访问后端，需要将 `localhost` 改成后端设备的局域网 IP，例如：

```javascript
const API_BASE = "http://192.168.1.10:8080/api";
```

## 8. 启动或打开前端

方式一：直接打开：

```text
frontend/index.html
```

方式二：在 `frontend/` 目录启动静态服务器：

```bash
cd frontend
python -m http.server 5500
```

访问：

```text
http://localhost:5500
```

局域网访问时将 `localhost` 替换为前端所在设备的 IP。

建议优先使用静态服务器方式访问前端，地址为：

```text
http://localhost:5500/index.html
```

前端页面包括：

```text
index.html      首页和商品搜索
register.html   注册
profile.html    登录、忘记密码、个人资料
publish.html    发布商品
detail.html     商品详情和留言
orders.html     订单管理
admin.html      管理中心
```

## 9. 验证安装

建议按以下顺序验证：

1. 首页能打开并显示商品列表。
2. 可按关键词或分类搜索商品。
3. 可注册用户并登录。
4. 登录后可发布商品。
5. 可进入详情页留言。
6. 可创建订单。
7. 可在订单管理页查看订单并更新状态。
8. 可在个人中心修改昵称、手机号、邮箱。
9. 连续 3 次输入错误密码后，账号会锁定 10 分钟。
10. 可在个人中心点击“忘记密码”，通过已注册邮箱验证码重置密码。
11. 使用管理员账号登录后，可进入管理中心禁用普通用户、删除留言、下架商品。

后端健康检查可访问：

```text
http://localhost:8080/api/actuator/health
```

如果返回 `UP`，说明后端服务已正常运行。

## 10. 常见问题

### 10.1 数据库连接失败

检查：

- MySQL 是否启动。
- `campus_secondhand` 数据库是否已创建。
- `application.yml` 中用户名和密码是否正确。
- MySQL 端口是否为 3306。

### 10.2 前端接口请求失败

检查：

- 后端是否启动。
- `frontend/assets/js/api.js` 中 `API_BASE` 是否正确。
- 浏览器 Network 面板中的请求地址是否指向正确 IP 和端口。

### 10.3 其他设备无法访问后端

检查：

- 两台设备是否在同一局域网。
- 后端设备防火墙是否允许 8080 端口。
- 前端 `API_BASE` 是否仍写为 `localhost`。

### 10.4 邮箱验证码发送失败

检查：

- `spring.mail` 是否配置为真实可用的 SMTP 服务。
- SMTP 用户名、密码、端口是否正确。
- 网络是否允许访问邮件服务器。

### 10.5 商品图片不显示

当前系统保存的是图片地址。请确认输入的图片地址可被浏览器访问，例如：

```text
assets/images/book.svg
```

### 10.6 启动时报表结构错误

后端配置为：

```yaml
spring:
    jpa:
        hibernate:
            ddl-auto: validate
```

因此后端不会自动创建或修改表结构。若启动时报缺少表或字段，请重新执行 `database/schema.sql`，必要时备份数据后重建 `campus_secondhand` 数据库。
